# report-pptx

`reports/{종목명}.md`(종목 리서치)를 읽어 **디자인된 PPTX**(`reports/{종목명}.pptx`)로 변환한다.

역할 분담:
- **md 해석·요약은 너(Claude)가** 한다 — md 구조가 제각각이어도 의미를 읽어 아래 SPEC(JSON)으로 정리.
- **렌더링은 `build_pptx.py`가** 한다 — 디자인 규칙(색·폰트·표 넘침 방지)은 전부 코드에 고정돼 있다. 디자인을 직접 손대지 말고 SPEC만 잘 채워라.

## 실행 절차

1. 대상 종목 확정. 인자로 종목명이 오면 `reports/{종목명}.md`를 읽는다. 종목명이 모호하거나 파일이 없으면 `reports/`를 확인하고 한 번만 되묻는다.
2. md 전체를 읽고 **7개 섹션 내용을 추출**해 SPEC(JSON)을 만든다. 아래 스키마를 따른다.
3. SPEC을 `reports/.spec_{종목명}.json`으로 저장한다(임시 파일, UTF-8).
4. 빌더 실행. 빌더는 **이 스킬 폴더의 `build_pptx.py`**다. 실행할 때 안내되는 `Base directory for this skill: <DIR>` 경로(설치 위치마다 다름)를 그대로 써라. 플러그인 환경이면 `${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/build_pptx.py`도 같은 파일을 가리킨다.
   ```
   # Windows에서 `python` 호출이 실패하면 `py` 런처를 쓴다(둘 중 되는 쪽).
   py "<Base directory>/build_pptx.py" "reports/.spec_{종목명}.json" "reports/{종목명}.pptx"
   ```
   경로에 공백/한글이 있으면 따옴표로 감싼다(PowerShell·bash 공통). 출력은 항상 현재 프로젝트의 `reports/` 아래에 만든다. 성공 시 `OK -> ...`가 출력된다.
5. 출력 경로를 사용자에게 알린다. 임시 spec json은 남겨도 되고 지워도 된다(원하면 정리).

## SPEC 스키마

모든 섹션은 선택적이다. 없으면 해당 슬라이드는 "자료 없음(근거 부족)"으로 렌더되며 **7장 순서는 항상 유지**된다.

```json
{
  "stock_name": "삼성전자",
  "ticker": "005930",
  "as_of": "2026-06-11",
  "created": "2026-06-11",
  "overall": "조정 우위 / 과열 경계",
  "overview": { "bullets": [ {"text": "...", "source": "출처: ..., 기준일 ..."} ] },
  "financials": {
    "headers": ["지표", "FY2023", "FY2024", "FY2025"],
    "rows": [ ["매출액", "...", "...", "..."], ["영업이익", "...", "...", "..."] ],
    "unit": "단위: 억원",
    "source": "출처: DART, 각 연도 사업보고서"
  },
  "price": {
    "metrics": [ {"label": "현재가", "value": "299,250원"}, {"label": "MA20", "value": "305,338원"} ],
    "change_rates": {"1주": -14.9, "1개월": 7.3, "3개월": 63.1},
    "price_series": [ {"date": "2026-01-02", "close": 71000} ],
    "comment": "추세 코멘트 한두 줄",
    "source": "출처: FinanceDataReader, 기준일 2026-06-11"
  },
  "sentiment": {
    "verdict": "긍정 (과열 경계 동반)",
    "issues": [ {"text": "...", "source": "매체/링크", "date": "2026-06-05"} ]
  },
  "risk": {
    "items": [ {"title": "변동성 함정", "detail": "...", "source": "출처: ..."} ],
    "monitoring": [ "트리거1 ...", "트리거2 ..." ]
  },
  "conclusion": { "headline": "두괄식 한 줄 결론", "basis": [ "근거1", "근거2" ] }
}
```

가격 슬라이드 차트 우선순위: `price_series`(라인) > `change_rates`(막대). 둘 다 없고 `ticker`가 있으면 빌더가 FinanceDataReader로 6개월 종가를 자동 시도한다(실패해도 안전). 실제 추세선을 원하면 `ticker`를 꼭 채워라.

## 콘텐츠 규칙 (SPEC을 채울 때 반드시 지킨다)

- **출처 없는 수치는 싣지 않는다.** md에서 출처·기준일이 확인되는 값만 넣는다. 출처를 못 찾은 값은 SPEC에서 제외하거나 텍스트에 "(근거 부족)"으로 명시한다. 추정치는 "추정"이라고 적는다.
- **매수·매도·목표가 단정 금지.** "매수하세요", "목표가 X원", "지금이 매수 타이밍" 같은 표현을 SPEC에 넣지 않는다. 상승 동력/과열, 촉매/리스크 등 **판단 근거까지만** 적는다.
- **문체는 "~입니다" 체.** 결론은 두괄식.
- **재무표는 행을 너무 많이 넣지 말 것**(빌더가 7행까지만 렌더, 초과분은 "…외 N행"으로 축약된다). 핵심 지표(매출액·영업이익·순이익) 위주로.
- 셀·본문 텍스트가 길면 빌더가 …로 자른다. 한 줄을 짧고 핵심만.
- DART 등에서 정합성이 의심되는 값(예: 분기 절대치 이상)은 SPEC에 넣지 말거나 "확인 불가"로 표기한다.

## 디자인 (코드 고정 — 참고용, 수정 금지)

- 포인트색 KB 옐로우 `#FFBC00`, 본문 그레이/화이트, 표지 잉크 배경. 차분한 금융 리포트 톤.
- 가드레일 자동 반영: 표지 첫머리 "무료 공개 데이터 기반 학습용" 한 줄 + 마지막 슬라이드에 데이터 출처·기준일 목록(spec의 source 필드에서 자동 수집). source를 빠짐없이 채울수록 출처 목록이 완전해진다.
- 한글 폰트 **맑은 고딕** 하나로 고정(라틴/EastAsian 모두 지정 → 글자 깨짐 방지).
- 표는 슬라이드 폭에 맞춰 컬럼 자동 분배 + 행 수 상한 + 텍스트 길이 제한으로 잘림 방지.
- 모든 슬라이드 하단에 "학습·연구 목적, 투자 권유 아님" 고지 + 페이지 번호.

## 의존성

- `python-pptx` 필요 (`pip install python-pptx`). 차트는 PPTX 네이티브 차트로 그려 matplotlib·한글 폰트 설정이 필요 없다.
- 선택: `FinanceDataReader`(ticker로 6개월 추세선 자동 생성 시).
- 한글 폰트는 **맑은 고딕**을 쓴다. Windows에는 기본 탑재. 비Windows에서 PPTX를 열 때 폰트가 없으면 시스템에 맑은 고딕(또는 대체 한글 폰트)을 설치하면 된다.
- Python 실행: `python build_pptx.py ...`가 기본. Windows에서 `python` 런처가 안 잡히면 `py`를 쓴다(둘 중 되는 쪽).

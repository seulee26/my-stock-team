`reports/$ARGUMENTS.md`를 품질 점검합니다. 입력 종목: **$ARGUMENTS**

`verification-analyst` 서브에이전트를 **Agent 도구로 호출**해 아래를 수행하게 하세요(이 에이전트는 **읽기 전용** — 리포트 파일을 직접 수정하지 않습니다).

- 대상: `reports/$ARGUMENTS.md` (없거나 종목이 모호하면 `reports/`를 확인하고 한 번만 되묻기)
- 점검 4축:
  - **정확성**: 수치가 데이터와 맞고 계산·단위 오류가 없는가 (의심 수치는 FDR/pykrx/DART로 스팟체크, 불가 시 "검증 불가")
  - **일관성**: 본문·표·결론이 서로 어긋나지 않는가 (risk 반론이 결론에 반영됐는가 포함)
  - **완결성**: 5부 섹션 + 분석 관점(기술적·뉴스심리·리스크·펀더멘털·과열점검)이 빠짐없이 담겼는가
  - **근거·형식**: 수치마다 (출처·연도)가 있고, 매수/매도 등 투자행동 단정 없이 양식(첫머리/말미 고지 포함)을 지켰는가
- 산출물: **문제 표**(# / 위치 / 축 / 무엇이 문제 / 어떻게 고칠지 / 심각도) + 축별 요약 + **통과 / 보류** 판정(사유 포함).

보류로 나오면, 헤드(당신)가 지적 사항을 반영해 `reports/$ARGUMENTS.md`를 수정한 뒤 재점검을 제안하세요. 검증 애널리스트는 고치지 않고 지적만 합니다.
